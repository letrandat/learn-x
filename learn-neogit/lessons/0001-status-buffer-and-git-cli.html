001: The Status Buffer — Where neogit meets git CLI

Goal: Open neogit, read the status buffer, and see the git CLI behind every action.

Open neogit

From any git repo, run: 

:Neogit

Or if you have a keybind, use that. You'll see the status buffer — neogit's main view.

The status buffer anatomy

The status buffer is an interactive git status. Sections, top to bottom:

SectionGit CLI equivalentWhat it shows
HEAD (Lines)
↳ Head:git log -1 --onelineYour current local branch, commit hash, and message
↳ Merge:git log -1 --oneline @{u}Your upstream remote tracking branch and its latest commit
Untrackedgit status --porcelain (??)Files git doesn't know about
Unstagedgit diffModified files not yet staged
Stagedgit diff --cachedFiles added to the index, ready to commit
Stashesgit stash listSaved work-in-progress snapshots
Unpushedgit log @{upstream}..HEADCommits you have that the remote doesn't
Unpulledgit log HEAD..@{upstream}Commits the remote has that you don't
Recentgit log --oneline -10Last 10 commits on this branch
💡 Pro-Tip: While Neogit runs git branch -vv behind the scenes to gather your branch state, it elegantly splits your local status (Head) and your remote status (Merge) into two lines. You can see this exact split on your CLI by running:
git log -1 --oneline && git log -1 --oneline @{u}
Core insight: Every section of the status buffer is output from a git CLI command. neogit isn't hiding git — it's rendering git's output in an actionable UI.

Two essential diagnostic keys

$ — Command History

Press $ in the status buffer. You'll see every git command neogit has run this session: exact CLI invocations, exit codes, and timing. This is your direct window into git CLI.

1234 — Depth Control

Press 1 to collapse all sections to top level (filenames only). 2 shows files and section headers. 3 shows hunks. 4 shows full diff. Each depth level corresponds to the detail you'd get from different git diff flags.

Exercise: Stage + Commit, watching the CLI

  1. Make a change to any file in a repo (or touch test.md for a new file)
  2. Open neogit: :Neogit
  3. Press $ — see git status and git diff in the history
  4. Close history (q), move cursor to your file, press s to stage
  5. Press $ again — see git add (or git add -N for new files)
  6. Close history, press c c to commit
  7. Write a message, save, close the commit buffer (:wq)
  8. Press $ — see git commit with your message
Compare with lazygit: In lazygit, git commands run invisibly unless you open a log panel. In neogit, $ shows every command by default. The philosophy: transparency builds understanding.

Quick quiz

Reveal Answer
