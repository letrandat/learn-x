003: Hunk and Line Staging — Surgical Git Precision

Goal: Master staging and unstaging individual hunks and specific lines in neogit, and see how they map to git CLI patch options.

The Power of Partial Staging

Imagine you have been working on a file for an hour. You have added a feature, updated the documentation, and inserted several console.log() statements for debugging.

If you stage the entire file, you commit your debug noise alongside the clean code. If you don't stage it, you can't commit the new feature. This is where partial staging shines: you can stage only the hunks or lines representing the feature, leaving the debug logs unstaged in your working tree.

Key concept: A hunk is a contiguous block of changed lines. By staging at the hunk or line level, you curate precise, readable commits.

Hunk Staging in neogit

In the status buffer, you can inspect and stage changes hunk-by-hunk.

1. Expanding file diffs

Move your cursor to an unstaged file in the status buffer and press Tab to expand it. You can also press 4 to expand all file diffs in the buffer at once.

2. Navigating between hunks

While you can navigate line-by-line using j and k, neogit provides quick jumps:

  • Press } to jump to the next hunk header.
  • Press { to jump to the previous hunk header.

3. Staging and unstaging a hunk

Position your cursor anywhere within a hunk (or on its header) and press:

  • s to stage the hunk.
  • u to unstage the hunk.

CLI equivalents: git add -p <file> (interactive staging) and git restore --staged -p <file> (interactive unstaging)

Line-level Staging: Ultimate Precision

What if a single hunk contains both a code improvement and a debug statement? You can stage down to the individual line.

  1. Move your cursor to the first line you want to stage.
  2. Press v (or V) to enter Visual mode.
  3. Move your cursor to highlight all the lines you want to select.
  4. Press s to stage only those highlighted lines.
  5. Press u if you want to unstage only the highlighted lines.
⚠️ Destructive discarding: You can select lines in Visual mode and press x to discard them. This permanently deletes them from your working tree. This maps to git restore -p <file>. Use with caution!

Behind the Scenes: How Git CLI does it

If you press $ after staging a hunk or selected lines, you won't see a simple git add command. Instead, you'll see neogit doing the heavy lifting:

  1. neogit generates a patch file representing the exact hunk or lines.
  2. neogit runs: git apply --cached --whitespace=nowarn and pipes the patch into it.

This bypasses the interactive prompt menu of the CLI's git add -p, giving you a faster, visual feedback loop.

Quick Quiz

Q1: Which keys jump between hunk headers in neogit status buffer?

Reveal Answer
Answer Q1: { and } — jumps to previous/next hunk headers.

Q2: To stage specific lines within a hunk, which mode must you enter first?

Reveal Answer
Answer Q2: Visual mode — select lines using v or V, then stage with s.
