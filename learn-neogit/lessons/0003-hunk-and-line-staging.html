003: Hunk Staging and Reversing Selected Lines

Goal: Master staging and unstaging individual hunks in neogit, and see how to reverse specific lines using visual selection.

The Power of Partial Staging

Imagine you have been working on a file for an hour. You have added a feature, updated the documentation, and inserted several console.log() statements for debugging.

If you stage the entire file, you commit your debug noise alongside the clean code. If you don't stage it, you can't commit the new feature. This is where partial staging shines: you can stage only the hunks representing the feature, leaving the debug logs unstaged in your working tree.

Key concept: A hunk is a contiguous block of changed lines. By staging at the hunk level, you curate precise, readable commits.

Hunk Staging in neogit

In the status buffer, you can inspect and stage changes hunk-by-hunk.

1. Expanding file diffs

Move your cursor to an unstaged file in the status buffer and press Tab to expand it. You can also press 4 to expand all file diffs in the buffer at once.

2. Navigating between hunks

While you can navigate line-by-line using j and k, neogit provides quick jumps:

  • Press } to jump to the next hunk header.
  • Press { to jump to the previous hunk header.

3. Staging and unstaging a hunk

Position your cursor anywhere within a hunk (or on its header) in normal mode and press:

  • s to stage the hunk under the cursor.
  • u to unstage the hunk under the cursor.

CLI equivalents: git add -p <file> (interactive staging) and git restore --staged -p <file> (interactive unstaging)

Line-level Reversing: Visual Mode Precision

While staging and unstaging operates on the hunk under the cursor, neogit allows line-level precision using the Reverse key (-).

If you have staged a hunk but want to revert specific lines in your working tree while keeping them staged in the index, you can use Visual mode:

  1. Move your cursor to the first line within the staged hunk you want to reverse.
  2. Press V to enter Visual Line mode.
  3. Highlight the specific lines you want to reverse.
  4. Press - to reverse only those selected lines.
Key difference: Staging and unstaging (s and u) apply to the file or hunk under the cursor in normal mode. Visual selection with V is used specifically with - to reverse individual lines within a hunk.

Behind the Scenes: How Git CLI does it

If you press $ after staging a hunk or reversing lines, you won't see a simple git add command. Instead, you'll see neogit doing the heavy lifting under the hood by generating patches:

  1. neogit generates a patch file representing the exact hunk or lines.
  2. neogit runs: git apply --cached --whitespace=nowarn for staging hunks, or git apply --reverse to reverse lines.

This bypasses the interactive prompt menu of the CLI's git add -p, giving you a faster, visual feedback loop.

Quick Quiz

Q1: Which keys jump between hunk headers in neogit status buffer?

Reveal Answer
Answer Q1: { and } — jumps to previous/next hunk headers.

Q2: To reverse specific lines within a staged hunk back into the working tree, which combination is used?

Reveal Answer
Answer Q2: Visual selection + - — enters Visual mode and reverses only the selected lines within the hunk.
