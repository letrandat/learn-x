# Mission: Neogit (and Git CLI)

## Why

Switching from lazygit to neogit for a tighter Neovim-integrated git workflow, while learning the actual git CLI commands that neogit runs underneath.

## Success looks like

- Perform daily git operations (stage, commit, push, pull, branch, rebase, stash) in neogit without reaching for lazygit
- Can translate any neogit action to its equivalent git CLI command
- Understand git's mental model (index, working tree, staging area) well enough to debug problems from the terminal
- Confident with git CLI for scripting and automation, not just interactive use

## Constraints

- New to Neovim (learning Neovim navigation alongside neogit)
- Already know git concepts (commit, push, pull, branch) through lazygit's UI, but not the CLI commands
- Want lessons that show the git CLI under every neogit action

## Out of scope

- Magit (Emacs) — neogit covers the same philosophy in Neovim
- Other Neovim git plugins (gitsigns, diffview, fugitive) — they complement neogit but aren't the focus
- Git internals (objects, refs, packfiles) — too deep for now
