snacks.nvim Glossary

Canonical language for this course. Every lesson uses these terms exactly. Add a term only once Dat can use it correctly.

Configuration model

snacks.nvim: A single Neovim plugin by folke bundling ~28 small "quality of life" modules (picker, terminal, dashboard, notifier, indent, scope…). LazyVim ships it as a core dependency.
Avoid: "the snacks plugins" (it is one plugin, many modules).

module: One feature inside snacks (e.g. picker, terminal, notifier). Configured by a top-level key in the opts table. A module is on only if you pass options for it or set enabled = true (LazyVim enables several by default).

opts (the merge model): The options table in a plugin spec. LazyVim deep-merges your opts on top of its own defaults, which sit on top of snacks' built-in defaults. Effective config = snacks defaults → LazyVim defaults → your opts. You only write the keys you want to change.
Avoid: "the config" (be specific: defaults vs. override).

Picker

picker: The snacks module that presents a searchable list (files, grep results, buffers, LSP symbols…) in a floating window with fuzzy matching. LazyVim's default picker — what <leader>ff opens.

source: A named picker type defining what to list and how (its finder + format + preview). Snacks has 40+. Invoked as Snacks.picker.files() or via a LazyVim keymap. Configured under picker.sources.<name>.
Avoid: "picker mode", "search type".

finder: The engine a source uses to generate items — e.g. the files source's finder shells out to fd (else rg/find); the grep source's finder shells out to ripgrep.

files source: Lists filenames under the root. Defaults: hidden=false, ignored=false, follow=false. Backs <leader>ff.

grep source: Live full-text search across files (ripgrep). live=true by default. Backs <leader>/.

Source options

ignored: When false (default), files matching .gitignore are excluded. Caveat: the backend (fd) only honours .gitignore inside a git repo; outside one, gitignored paths leak in.

hidden: When false (default), dotfiles/dotdirs are excluded.

follow: When true, the finder follows symbolic links.

exclude: An array of glob patterns the finder always drops (e.g. { "node_modules", ".git" }). Works regardless of git status — the reliable hammer.

cmd / args: Force the backend command ("fd"/"rg"/"find") and pass extra CLI args. Auto-detected if omitted.

Layout

layout: The on-screen arrangement of a picker's windows (input box, results list, preview). Chosen by a preset (snacks switches between default horizontal and vertical based on terminal width).

width / height: A value in 0–1 is a proportion of the screen; an integer ≥1 is a character count; 0 means full. So 0.9 = 90% of the screen.

Terminal

terminal module: snacks' floating/split terminal. Snacks.terminal.toggle(cmd, opts) shows/hides a terminal keyed by command+cwd+env+count. Styling comes from win (a snacks.win.Config): with no cmd it defaults to a bottom split, with a cmd it floats.
