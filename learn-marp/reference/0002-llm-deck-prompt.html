LLM Deck-Generation Prompt Kit

Copy-paste guardrails for driving a weak model from notes → Marp → PPTX. Pairs with Lesson 0002.

A · The rules block (paste once, up front)

Give the model the guardrails before it writes any Markdown. This pre-empts the four classic failures.

You are generating a slide deck in Marp Markdown. Follow these rules exactly:

STRUCTURE
- The file MUST begin with YAML front-matter containing `marp: true`.
- Put deck-wide settings in that front-matter: theme, paginate, size.
- Separate every slide with a `---` ruler on its own line.

THEME & DIRECTIVES
- theme must be one of: default, gaia, uncover. Never invent a theme name.
- Use ONLY real directives: theme, paginate, header, footer, class,
  backgroundColor, backgroundImage, color, size, math.
- Do NOT invent directives (no `columns`, `layout`, `transition`, etc.).
- A directive for ONE slide only must start with an underscore, e.g.
  `<!-- _class: lead -->`. Without the underscore it applies to that
  slide AND every slide after it.

IMAGES
- Background images use the `bg` keyword: `![bg](url)`, `![bg cover](url)`,
  `![bg left:40%](url)`. A plain `![](url)` is a small inline image, not a bg.

CONTENT BUDGET
- Max ~6 bullets or ~40 words per slide. If more, split into two slides.
- Marp does NOT reflow overflow — content that doesn't fit is cut off.

OUTPUT
- Output ONLY the Markdown, in one code block. No commentary.

B · Stage prompts (drive it in turns)

Stage 2 · context 
Here are my notes and code: <paste>.
Summarize the key points an audience of <who> needs.
Ask me up to 5 clarifying questions before we plan any slides.
Stage 3 · outline (lock this) 
Draft a slide-by-slide OUTLINE only — no Markdown.
One line per slide: title + a one-sentence purpose.
Target ~<N> slides with a clear arc (hook → problem → solution → proof → ask).
Wait for my approval before writing any Marp.
Stage 4 · generate 
Convert the approved outline into Marp Markdown.
Apply the RULES BLOCK above. theme: <gaia>, size: 16:9, paginate: true.
Use `<!-- _class: lead -->` on the title slide only.
Stage 6 · targeted fix 
Slide <n> <describe the visible bug>.
<name the cause from the checklist>. Fix only that slide; return the full file.

C · Review checklist (you, in the preview)

  • ☐ File starts with marp: true front-matter
  • ☐ Theme is default / gaia / uncover (not invented)
  • ☐ No made-up directives
  • ☐ One-slide styles have the leading _ (no leaking)
  • ☐ Backgrounds use the bg keyword
  • ☐ No slide overflows / text cut off
  • ☐ Slide count ≈ the locked outline

Full symptom→fix table in reference 0001, §8.

D · Export commands

# preview live while editing
marp -p -w deck.md

# PPTX as images (pixel-perfect, NOT editable) — keeps presenter notes
marp deck.md --pptx

# PPTX editable text/shapes (experimental; needs LibreOffice; lower fidelity; no notes)
marp deck.md --pptx --pptx-editable

# also: --pdf , --images png
# All of these need Chrome/Edge/Firefox installed.

E · When the model is stuck — paste a doc at it

  • Scope/directive confusion → marpit.marp.app/directives
  • Image/background bugs → marpit.marp.app/image-syntax
  • Theme/math/fit → github.com/marp-team/marp-core
  • Export/PPTX → github.com/marp-team/marp-cli
