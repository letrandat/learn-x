Marp Cheat Sheet

Markdown → slides. The reference you keep open while reviewing an LLM's output.

1 · Minimal valid deck

---
marp: true
theme: default
paginate: true
---

# Title slide

Body text here.

---

## Second slide

- bullet
- bullet
#1 rule: the file MUST start with front-matter containing marp: true. Without it, Marp tools treat the file as plain Markdown — no slides. This is the single most common thing an LLM forgets.

2 · Slide separation

A horizontal rule splits slides: --- (also ***, ___). The first two --- rulers delimit the YAML front-matter; every --- after that is a slide break.

3 · Directive scopes most confused

ScopeWritten asApplies to
GlobalIn front-matter, or <!-- theme: gaia -->Whole deck
Local<!-- paginate: true -->This slide and all after
Spot (underscore)<!-- _class: lead -->This one slide only
The leading _ = "this slide only." LLMs constantly drop the underscore (leaking a style to every later slide) or add one where it shouldn't be. If a style "bleeds" down the deck, look for a missing _.

4 · Directive reference

Global (deck-wide)

marpmust be true
themedefault · gaia · uncover
size16:9 or 4:3
mathkatex / mathjax
styleinline CSS override
headingDividerauto-split on heading level

Local / spot

paginatetrue·false·hold·skip
headertop text
footerbottom text
classe.g. lead, invert
backgroundColorCSS color
colortext color

Themes are: default, gaia, uncover. If an LLM writes theme: gaudy or any other name → wrong, it'll silently fall back.

5 · Images & backgrounds

Resize a normal image

![w:300](cat.jpg)          # width 300px
![w:200 h:150](cat.jpg)    # both
![width:50%](cat.jpg)

Background images (the bg keyword)

![bg](photo.jpg)              # full-slide background
![bg cover](photo.jpg)        # fill (default)
![bg contain](photo.jpg)      # fit inside, no crop  (alias: fit)
![bg left](photo.jpg)         # image on left half, text on right
![bg right:40%](photo.jpg)    # right 40%, text on remaining 60%
![bg left](a.jpg)
![bg right](b.jpg)            # two-up split

![bg blur:4px brightness:.7](photo.jpg)   # filters
Watch for: an LLM using plain ![](bg.jpg) without the bg keyword (gives a tiny inline image, not a background), or mixing up cover (crops to fill) vs contain/fit (letterboxes).

6 · Marp-Core extras

Fit heading# <!-- fit --> Big Title auto-scales to slide width
Math$E=mc^2$ inline · $$ ... $$ block (needs math: directive)
Emoji:smile: → 😄
ColumnsNo native syntax — done via style CSS or HTML <div>. LLMs often hallucinate a column directive that doesn't exist.

7 · Export (Marp CLI)

npx @marp-team/marp-cli@latest deck.md            # → deck.html
marp deck.md -o out.pdf                           # PDF  (needs Chrome/Edge/Firefox)
marp deck.md --pptx                               # PowerPoint
marp deck.md --images png                         # one PNG per slide
marp -w deck.md                                   # watch mode
marp -p deck.md                                   # preview window

Or just paste markdown into the no-install web editor: web.marp.app — best feedback loop.

8 · LLM failure-mode checklist

SymptomLikely cause / fix
Renders as one long page, no slidesMissing marp: true front-matter
Style bleeds to all later slidesUsed class: instead of _class: (missing underscore)
Theme ignoredInvalid theme name — must be default/gaia/uncover
Background is a tiny imageMissing bg keyword in ![bg](...)
Image cropped weirdlycover vs contain/fit mixup
Made-up directive (e.g. columns:, layout:)Hallucination — Marp has no such directive; use CSS via style
Comment shows on slideHTML comment used for prose, but Marp comments are directives; for notes use presenter notes
Slide overflows / text cut offToo much content per slide; split it. Marp does not auto-paginate overflow.
