# Mission

## What Dat wants to learn

How to **understand and use Matt Pocock's skills system** (`mattpocock/skills`) — starting
from `setup-matt-pocock-skills` and expanding outward to the full engineering workflow.

## Why (the grounding)

Dat is adopting agent skills as a real engineering practice — not vibe coding. The aim is to
wield the skills fluently in his own repos: know which skill to reach for, why it exists, and
how they compose, so he can adapt/fork them.

## What "done" looks like (current read — confirm with Dat)

- Can explain the user- vs model-invoked axis and the four failure modes from memory.
- Can run `setup-matt-pocock-skills` on a repo and read the three configs it produces.
- Can trace a feature through the daily flow (grill → prd → issues → tdd → maintain).
- Eventually: writes/forks a skill of his own.

## End goal (confirmed 2026-06-19)

**Both, in that order:** master *using* Matt's skills as-is first, then graduate to
*authoring* his own skill library in this style. Sequence lessons accordingly —
usage flow before `writing-great-skills`.
