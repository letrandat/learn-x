Lesson 0006 · The Router Over Everything

ask-matt: the map of the whole system

You've learned the skills one at a time. ask-matt is the skill that holds them all as a single map — so when you forget which one to reach for, you ask. This lesson is mostly consolidation: every skill you know, placed on one route, with the gaps named.

Why this skill exists "You don't remember every skill, so ask." ask-matt stores no process and ships no code — it's a router: a description of how the user-invoked skills connect. Marked disable-model-invocation: true, so the model never fires it on its own. You invoke it, on purpose, when lost.

One new word: flow

A flow is a path through the skills. The system is organised as:

  • One main flow — idea → ship, the route most work travels.
  • Two on-ramps — situations that generate work, then merge onto the main flow.
  • Standalone skills — off the main flow entirely.

That's the whole organising idea. Everything you've learned hangs off it.

So the "daily flow" from Lessons 2–5 is the main flow plus one on-ramp. ask-matt didn't add skills — it gave them a shape.

The one rule the map enforces: context hygiene

Keep steps 1–3 in one unbroken context window — don't /compact or clear until after /to-issues — so grilling, PRD, and issues all build on the same thinking. Each /implement then starts fresh from its issue.

Why: the smart zone The window (~120k tokens on SOTA models) where the model still reasons sharply. If a session nears it before /to-issues, don't push on a degraded model — /handoff and continue fresh. This is why the issues are made independent: so each build can afford a clean context.

What's NOT on this map (and why)

Two skills you asked to learn are deliberately absent — and the reason teaches you something:

  • prototype — it is on the map, but only as a detour off step 1, reached via /handoff when a question needs a runnable answer. Next lesson.
  • diagnosing-bugs — absent entirely. Why? ask-matt routes only the user-invoked skills. diagnosing-bugs is model-invoked (no disable-model-invocation) — it fires when you describe a bug, you don't navigate to it. It's a reflex, not a destination. (Recall the user/model axis from Lesson 1.)

That's the deep cut: a router only lists the doors you open. The reflexes that fire on their own don't need a map.
