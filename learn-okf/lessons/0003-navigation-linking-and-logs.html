Lesson 0003 • Open Knowledge Format

Navigation, Linking, and Logs

Connecting concepts, structural navigation, and chronological changelogs.

Connecting Concepts: Relative Linking

In OKF, relationships between concepts are represented using standard relative markdown links. Instead of storing complex database IDs or global unique keys inside the document body, you link documents using their filesystem paths.

For example, if a document at projects/wiki-init.md wants to reference a runbook at runbooks/deploy.md, it simply uses: 

Refer to the [Deploy Runbook](../runbooks/deploy.md) for more details.

An AI agent reading these files parses these relative links to construct a directed relationship graph of the entire knowledge base.

Directory Listings: index.md

An index.md file can reside in the root or any subdirectory of an OKF bundle. Its purpose is to provide a directory listing that lists subdirectories and concepts.

This enables progressive disclosure: an AI agent or consumer tool can navigate the directory tree level-by-level, reading the index summaries to discover what's available without having to ingest the entire corpus into its context window at once.

An index.md typically contains list items with a link and a short description: 

# Projects directory index

* [Wiki Init](wiki-init.md) - Personal and professional wiki initialization project.
* [Web App](../apps/web-app.md) - The main client-facing dashboard application.

Chronological Logs: log.md

A log.md file records the update history of the bundle. Key specifications for `log.md` include:

  • Date grouping: Update entries must be grouped by date headings formatted as ISO-8601 YYYY-MM-DD.
  • Newest first: The log must be ordered chronologically with the newest entries at the top.
  • Prose entries: Changes are written in clear text, often starting with a bold action verb (e.g., **Creation**, **Update**, **Deprecation**).

Permissive Consumption

A key principle of the OKF standard is Permissive Consumption. This requires that tools parsing an OKF bundle must be forgiving. They should not crash or reject the bundle if they encounter:

  • Broken relative links (missing files)
  • Unknown custom frontmatter keys or concept types
  • Missing recommended (but optional) metadata fields
