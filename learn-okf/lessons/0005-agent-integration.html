Lesson 0005 • Open Knowledge Format

Local Agent Integration

How to configure and empower AI agents to read, write, and maintain your OKF Wiki.

Enrichment vs. Consumption Agents

An OKF bundle is designed to be highly interoperable, acting as a shared memory layer between human owners and AI agents. In practice, agents interacting with an OKF bundle fall into two categories:

  • Consumption Agents: Read existing concepts to retrieve domain-specific context (e.g. querying a playbook before executing a task).
  • Enrichment/Maintenance Agents: Programmatically create, edit, and keep metadata up to date, maintaining chronological changelogs in log.md.

The Read-Write Agent Loop

To build a reliable agent integration, the agent needs clear rules on how to browse and update the wiki without corrupting its structure.

1. Navigating via Progressive Disclosure

Instead of dumping the entire repository into the context window, consumption agents should read index.md first. This acts as a table of contents. If they need details on a specific project or domain, they follow the link to that file (or sub-directory index) dynamically. This minimizes token consumption and scales to thousands of documents.

2. Writing Concepts with Structural Integrity

When an enrichment agent creates or edits a concept document, it must follow these constraints:

  • Always include YAML frontmatter: It must start and end with --- on its own line and include a valid type value.
  • Compute Concept IDs: The agent should match relative links correctly (e.g. [label](../subdir/target.md)) when linking concepts.
  • Use ISO-8601 timestamps: Record the modification time in the frontmatter timestamp field.

3. Maintaining the Changelog

Every time an agent modifies or adds a file, it must append a record to log.md at the root. The update entry must:

  1. Be grouped under a date heading formatted as ## YYYY-MM-DD. If the heading for today doesn't exist, create it at the top (right below the main title).
  2. Begin each change bullet point with a bold action verb (e.g., * **Creation** of `projects/new-concept.md` ...).
Model Context Protocol (MCP) Integration

In professional setups, agents are given access to the filesystem via the Model Context Protocol (MCP) filesystem server. This allows them to read, search, and edit local files directly via structured tools rather than relying on raw terminal commands.

Interactive Check

Practical Exercise

Let's write a simple Python automation script to simulate an Enrichment Agent! This script will automatically create a new concept document, update the root index, and write to the changelog.

Create a script named agent_simulate.py at the root of your workspace directory with the following contents: 

import os
from datetime import datetime, timezone

# Define paths
ROOT_DIR = os.path.dirname(os.path.abspath(__file__))
CONCEPT_PATH = os.path.join(ROOT_DIR, "projects", "agent-collab.md")
INDEX_PATH = os.path.join(ROOT_DIR, "index.md")
LOG_PATH = os.path.join(ROOT_DIR, "log.md")

# 1. Create the new concept file
concept_content = f"""---
type: project
title: Agent Collaboration Wiki
description: Integrating automated agents to collaborate on the OKF knowledge base.
tags: [agent, automation]
timestamp: {datetime.now(timezone.utc).isoformat().replace("+00:00", "Z")}
---

# Agent Collaboration Wiki

This concept was created automatically by our local simulation agent.
It links back to our [Initial Wiki Project](wiki-init.md).
"""

os.makedirs(os.path.dirname(CONCEPT_PATH), exist_ok=True)
with open(CONCEPT_PATH, "w", encoding="utf-8") as f:
    f.write(concept_content)
print(f"Created: projects/agent-collab.md")

# 2. Update root index.md
with open(INDEX_PATH, "r", encoding="utf-8") as f:
    index_lines = f.readlines()

new_index_entry = "* [Agent Collaboration Project](projects/agent-collab.md) - Integrating automated agents to collaborate on the OKF wiki.\n"
if not any("agent-collab.md" in line for line in index_lines):
    # Find list section and append
    index_lines.append(new_index_entry)
    with open(INDEX_PATH, "w", encoding="utf-8") as f:
        f.writelines(index_lines)
    print("Updated: index.md")

# 3. Update log.md
today = datetime.now().strftime("%Y-%m-%d")
with open(LOG_PATH, "r", encoding="utf-8") as f:
    log_content = f.read()

log_lines = log_content.splitlines()
entry_heading = f"## {today}"
log_entry = f"\n{entry_heading}\n\n* **Creation** of `projects/agent-collab.md` via agent simulation.\n* **Update** of `index.md` listing the new Agent Collaboration concept."

# Check if today's header is already there
if entry_heading in log_content:
    # Append bullet points below the header
    for idx, line in enumerate(log_lines):
        if line.strip() == entry_heading:
            log_lines.insert(idx + 2, "* **Creation** of `projects/agent-collab.md` via agent simulation.")
            log_lines.insert(idx + 3, "* **Update** of `index.md` listing the new Agent Collaboration concept.")
            break
    with open(LOG_PATH, "w", encoding="utf-8") as f:
        f.write("\n".join(log_lines) + "\n")
else:
    # Insert new section below title (first line)
    title_line = log_lines[0]
    remaining = "\n".join(log_lines[1:])
    new_log = f"{title_line}\n{log_entry}\n{remaining}"
    with open(LOG_PATH, "w", encoding="utf-8") as f:
        f.write(new_log)
print("Updated: log.md")

Now, perform the following steps:

  1. Run the simulation script in your terminal: 
    python3 agent_simulate.py
  2. Re-run the visualization script to compile the new relationship: 
    python3 visualize.py
  3. Open wiki_graph.html in your browser to verify that the new Agent Collaboration Wiki node is visible and linked!
