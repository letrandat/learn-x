OKF Folder Structure and Frontmatter

Understanding reserved files, Concept IDs, and metadata keys.

Bundle Structure

An OKF Bundle is simply a directory tree of UTF-8 markdown files. The folder structure itself is independent of the domain—you can organize concepts in whatever subdirectory hierarchy makes sense for your personal or professional wiki.

Reserved Filenames

The OKF specification reserves two filenames at any level of your wiki hierarchy. These files serve specific structural roles and must not be used to represent regular concept documents:

Filename Purpose
index.md Used to list directory contents to enable progressive disclosure as an agent navigates the hierarchy.
log.md Used to maintain a chronological log/history of changes and updates to the concepts in that folder.
Concept IDs

Every regular markdown file in the bundle represents a single Concept. A concept's identity is defined by its Concept ID, which is its file path relative to the bundle root, with the .md extension removed.

For example: A document stored at projects/my-website.md has the Concept ID projects/my-website.

Concept Documents: Frontmatter Specification

Each concept document consists of a YAML frontmatter block at the top and a markdown body.

Here is the standard schema for the frontmatter block: 

---
type: <Type name>                  # REQUIRED (e.g. project, playbook, runbook, metric)
title: <Optional display name>      # Recommended display title
description: <Optional summary>     # Recommended one-sentence summary
resource: <Optional canonical URI>  # URI for physical assets (e.g., code repos, databases)
tags: [<tag>, <tag>, ...]           # Optional array of classification tags
timestamp: <ISO 8601 datetime>     # Optional last-modified time
---
# Body goes here...
Producer/Consumer Independence

Aside from type, all other frontmatter keys are optional. You can add custom keys (like owner or status). An agent reading this wiki (the consumer) must tolerate unknown types and keys gracefully, treating them as generic concepts.
