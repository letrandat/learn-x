Parsing and Visualizing OKF

How consumer tools read frontmatter, build graphs, and render bundles.

The Consumer Pattern

An OKF bundle is designed to be parsed easily by simple script engines and agent compilers without any specialized database. A typical parser traverses the directories, scans concept files, and creates an in-memory graph representation.

Step 1: Discovering Concepts

The parser walks the filesystem, identifying files ending in .md. It skips system folders (like .git) and reserved file names (like index.md and log.md). The relative path of each file becomes its Concept ID.

Step 2: Extracting Metadata

For each concept, the parser reads the top YAML frontmatter block. It maps the standard keys:

  • type: Used to categorize node types (e.g. project, playbook).
  • title: Used as the node display name.
  • description: Used for previews or hover cards.

Step 3: Graph Linkage

The parser scans the markdown body of each file using a regular expression or a markdown AST parser to extract relative links like [label](../target/concept.md). It normalizes these paths relative to the current file location to compute target Concept IDs. An edge is created between the source and target concepts.

Your Local Visualizer: visualize.py

To make this process concrete, I have written a zero-dependency script at the root of your workspace named visualize.py. This script:

  1. Finds all concepts (like projects/wiki-init.md).
  2. Parses their frontmatter and markdown links.
  3. Compiles them into a single, beautiful, interactive HTML visualizer (wiki_graph.html) powered by D3.js.

Practical Exercise
